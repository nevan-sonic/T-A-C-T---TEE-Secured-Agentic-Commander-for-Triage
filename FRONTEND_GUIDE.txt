============================================================
  T.A.C.T. — FRONTEND REDESIGN GUIDE
  For: Frontend Developer
  Backend: DO NOT TOUCH  |  Port: 3000
============================================================

This file documents every API endpoint, the data shapes they
return, and everything you need to replace public/index.html
with a redesigned frontend while keeping the backend working.

------------------------------------------------------------
FOLDER STRUCTURE (what you care about)
------------------------------------------------------------

  public/
    index.html        ← THE ONLY FILE YOU NEED TO EDIT
                        (all frontend lives here — one file)

  server.js           ← backend, do not touch
  src/                ← backend TypeScript, do not touch
  dist/               ← compiled backend, do not touch

You replace the contents of public/index.html with your own
design. The server serves everything in public/ as static files,
so you can also add separate .css, .js, or image files inside
public/ if you want to split things up.

------------------------------------------------------------
RUNNING THE SERVER (backend stays running, you just edit HTML)
------------------------------------------------------------

  1. Follow REQUIREMENTS.txt to set up the project
  2. Start the server:   node server.js
  3. Open browser:       http://localhost:3000
  4. Edit public/index.html and hard-refresh (Ctrl+Shift+R)

------------------------------------------------------------
CRITICAL: SIGNING SETUP (do this on page load)
------------------------------------------------------------

The approval flow requires the browser to have a wallet key
that matches the server's configured approver DID.

On page load, do this SYNCHRONOUSLY (use XMLHttpRequest, NOT
fetch, so the key is ready before you do anything else):

  // 1. Sync the signing key from the server
  const xhr = new XMLHttpRequest();
  xhr.open("GET", "/api/dev-wallet", false); // false = synchronous
  xhr.send();
  const devData = JSON.parse(xhr.responseText);

  let signingKey = localStorage.getItem("t3_developer_private_key");
  if (devData.privateKey && devData.privateKey.length === 66) {
      signingKey = devData.privateKey;
      localStorage.setItem("t3_developer_private_key", signingKey);
  }
  if (!signingKey || signingKey.length !== 66) {
      signingKey = ethers.Wallet.createRandom().privateKey;
      localStorage.setItem("t3_developer_private_key", signingKey);
  }
  const signingWallet = new ethers.Wallet(signingKey);

  // 2. Register the DID with the server (seeds enclave secrets)
  fetch("/api/register-active-did", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ did: "did:t3n:" + signingWallet.address.toLowerCase().replace("0x","") })
  });

You need ethers.js v6 for signing. Load it from CDN:
  <script src="https://cdnjs.cloudflare.com/ajax/libs/ethers/6.7.0/ethers.umd.min.js"></script>

════════════════════════════════════════════════════════════
 6.  POST /api/approve
════════════════════════════════════════════════════════════
Submits a cryptographic signature to approve a pending action.

⚠ IMPORTANT: The signature must be from the wallet that
  matches app.approverDID exactly. Use the signing setup
  in the SIGNING SETUP section above.

How to sign and submit:

  async function signAndApprove(approvalId, approverDID) {
      // Build the message (MUST match this exact format)
      const message = [
          "T3 Agent Authorization Grant",
          "Agent DID: did:t3:agent:department-of-incidents",
          "Contract: z:system:incident-contracts",
          `Function: incident-${approvalId}`,
          `Approval ID: ${approvalId}`
      ].join("\n");

      // Sign with the synced wallet
      const signature = await signingWallet.signMessage(message);

      // Submit
      const res = await fetch("/api/approve", {
          method: "POST",
          headers: { "Content-Type": "application/json" },
          body: JSON.stringify({ id: approvalId, signature })
      });
      return res.json();
      // Success: { "status": "approved" }
      // Failure: { "error": "Invalid signature proof. Address recovery failed." }
  }

════════════════════════════════════════════════════════════
 8.  GET  /api/telemetry-metrics
════════════════════════════════════════════════════════════
Returns real-time service health from the simulated SRE service.
Poll every 500ms for a live chart.

Response:
  {
    "latency":   86,    // avg p99 latency in milliseconds
    "errorRate": 12     // error rate as integer percentage (0-100)
  }

When errorRate >= 45% for 4+ seconds, the backend auto-creates
an incident. This is what drives the live telemetry chart.

------------------------------------------------------------
POLLING STRATEGY (recommended intervals)
------------------------------------------------------------

  /api/telemetry-metrics    every 500ms   (live chart)
  /api/incidents            every 2000ms  (incident list)
  /api/approvals            every 1500ms  (approval panel)
  /api/ledger               every 3000ms  (audit log)

------------------------------------------------------------
INCIDENT STATUS FLOW (for status badges)
------------------------------------------------------------

  SRE Incident:
  Analyzing → Reading Logs → Classifying Severity →
  Creating PR → Awaiting Approval → Merging → Resolved
                                  ↓ (if rejected)
                                  Rolled Back

  CVE:
  Reading Vulnerable File → Analyzing CVE → Generating Fix →
  Validating Patch → Creating PR → Awaiting Approval →
  Merging → Canary Window → CVE Resolved

  Runbook:
  Fetching Runbook → Executing Steps → (steps may need approval) →
  Runbook Completed

  Cost Anomaly:
  Analyzing Cost Data in TEE → Cost Analysis Complete →
  Executing Remediations → Cost Anomaly Resolved

------------------------------------------------------------
TRIGGER TYPES (for icons / labels)
------------------------------------------------------------

  "github-cve"   → shield / security icon
  "pagerduty"    → bell / alert icon
  "cloudwatch"   → cloud / dollar icon
  "datadog"      → dog icon / monitor
  "prometheus"   → chart / fire icon
  "auto"         → robot / auto icon (self-detected by monitor)
  "manual"       → hand / user icon

------------------------------------------------------------
NOTES
------------------------------------------------------------

- All timestamps in the API are epoch milliseconds (number).
  Convert with: new Date(timestamp).toLocaleTimeString()

- All costs in costSaving are strings (e.g. "300150" = $300,150).
  Format as: "$" + parseInt(costSaving).toLocaleString()

- The backend has no WebSocket — use polling.

- The ethers.js version in use is v6 (not v5).
  CDN: https://cdnjs.cloudflare.com/ajax/libs/ethers/6.7.0/ethers.umd.min.js

- Do NOT remove the /api/dev-wallet sync logic — it is what
  makes approvals work in development without MetaMask.

- The "Confirm & Sign" button must call signAndApprove() using
  the approval's id and approverDID from /api/approvals.
